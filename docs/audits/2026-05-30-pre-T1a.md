- `phase-T0-report.md` uses keys `{phase, title, status, started, finished, min_goal_met, max_goal_met}`.
- The six Maquette reports (`phase-0-report.md` … `phase-3.5-report.md`) use a *different* schema: `{phase, status, min_met, max_met, duration_planned_days, duration_actual_days}` — no `id`/`title`, and `min_met`/`max_met` instead of `min_goal_met`/`max_goal_met`.

But the report frontmatter schema is **not standardized** — the Touch-era report (T0) and the Maquette-era reports disagree on key names for the same concepts (`min_met` vs `min_goal_met`). Any tool reading report frontmatter by key (e.g. `/pm-status`) will mis-read one set. Recommend standardizing the report template's keys.
