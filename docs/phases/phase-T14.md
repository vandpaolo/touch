# Phase T14 — Multi-file project model

- **Goal:** A "project" is a folder of related `.touch` files (preludes assemblies). The tree handles many files; cross-file references are v0.2 work.
- **Min:** Multi-file open in tree; per-file undo state isolated; recent-project history.
- **Max:** Project-level settings file (`.touch-project.json`); per-project provider override.
- **Exit criterion:** open 5+ `.touch` files in one project; edit each; saves remain isolated and correct.
