# Blockers

Mid-phase obstacles that require a re-design round.

Filename: `YYYY-MM-DD-slug.md`. Written by `/pm-blocker`. YAML frontmatter
holds `status: open | resolved`, `severity`, and `re_entry` (which design
layer needs to re-open).

A blocker doc must be **closed** (status flipped to `resolved`, resolution
section filled, links to updated docs added) before the phase it blocks
can resume `in_progress`.
